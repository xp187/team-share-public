用例结构与新建

用例目录解剖（以 t_8b1lane 为例）

文件作用
t_8b1lane.svtest class：创建 isprx_env_cfg，填入 user_def.sv 宏（HACT/HBP/VTOTAL/FPS/PORT_NUM/FRAME_NUM/TBK1/CHIP_SEL/MUX/DV…），设 4 个 *_check_on 开关，config_db 下发 env_cfg 和 default_sequence
user_def.sv本例全部时序/分辨率宏
cfg_frame0~8.txt每帧寄存器值（如 POL 0、CH_SEL 1、TEST_PRTCT 01110010，值为二进制字符串），由 env_cfg 打包进 CTRL_F/L
pattern/frame_0..7.ppm输入图像（P3 格式 PPM），常是指向 ppm_mix_*/ 的 symlink
test_lib.sv / test.finclude 本例 .sv / 编译文件列表（user_def + waves_dumper + incdir）
waves_dumper.sv用例级自定义波形抓取（makefile CASE_DESIDE 时优先）
generate_define.pl辅助生成宏定义

各用例的 t_xxx_vseq extends base_vseq 通常只调 super.body() —— 用例差异全在 test class 配置和 cfg_frame/pattern 文件。

新建用例流程

图 10：create_new_case.sh —— 复制模板后按用例名自动改参数；建完把用例名加入 case_list.txt 即进回归

用例名参数解析规则

名字字段作用目标说明
H3840 / H2560 / H1920 / H1440、V16、FPS120、PORT_NUM12user_def.sv时序与分辨率宏
CH_SELn / SDC / POLC / H4INV / POL2 / SD_CHOP / SCREN / DPLC_MODE / AVE_*cfg_frameN.txt寄存器字段
ratio240cfg_frameN.txt自动拆成 AVE_R/G/B
DPLC_MODE 1test class 开关关 digital/data_merge 比对，开 analog_check_on + digital_dplc_check

用法：cd top/tests && bash create_new_case.sh <新名>（或传 .txt 批量生成）。
