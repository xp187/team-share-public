run_tc.sh 用法

# RTL 仿真
run_tc.sh TOP       t_8b1lane chip_tb_top random clean
# 带覆盖率
run_tc.sh TOP_COV   t_8b1lane chip_tb_top random clean
# 网表（无前仿 SDF）/ 后仿（带 SDF）
run_tc.sh TOP_NETLIST      ...
run_tc.sh TOP_NETLIST_POST ...
# 看波形 / 覆盖率
run_tc.sh TOP     t_8b1lane chip_tb_top verdi
run_tc.sh TOP_COV t_8b1lane chip_tb_top cov / cov_rpt

前置条件：配好 $PROJECT_PATH/$DV_TOP_PATH/$DV_SCRIPT_PATH/$SIM_PATH 环境变量。seed 取 date +%N，随后调 make DEFINE_TOP=true TESTS_NAME=<case> CHIP_TOP_NAME=chip_tb_top SEED=$SEED。

makefile target

target说明
all（默认）comp + run
compVCS 编译：用例 test.f + user_def.sv + test_tcon.c + RTL 列表 HK1V11.f + tb.f + chip_tb_top；带 -cm line+fsm+tgl+cond+assert+branch -cm_hier hier.cfg
runsimv：+UVM_TESTNAME=<case> +FILE_PATH=<用例目录> +SIM_TEST_PATH=<仿真目录>
verdi / verdi_daidir波形调试
cov_dve / cov_verdi / cov_rpt覆盖率查看 / urg 报告
clean清理编译产物

批量回归

  • top/tests/run_case.py：逐 case bsub -Ip run_tc.sh TOP <case> chip_tb_top random clean，可传用例名或 case_list.txt。
  • run_parallel.py / run_muticase_script：并发版本。
  • 回归后跑 check_sim_results.sh 汇总 PASS/FAIL/SKIPPED。

要点

  • top/regression/hier.cfg 仅两行：+tree chip_tb_top.u_HK1V11.u_HK1V11_digital_top 0 / -tree chip_tb_top 1。
  • 单用例查看：run_tc.sh TOP_COV <case> chip_tb_top cov（verdi -cov）或 cov_rpt（urg）。
  • merge_coverage.py 的 project_name/user_names 仍是旧项目硬编码（HV1V21/zjbi），用前必须改成 HK1V11 和自己的路径。
  • 回归无统一调度脚本：case_list.txt（41 个用例）+ run_case.py（bsub 逐 case）是最常用方式。
