概览与验证思路

环境定位

本环境是显示驱动芯片 HV2M23 的 TCON 方向模块验证环境（DV_TCON_C），DUT 为整颗芯片（pad 级）。仿真在 Linux 服务器上执行（VCS + UVM-1.1），本地仅代码副本。

项目说明
环境根目录DV_TCON_C/（Linux 服务器路径 /XXW_PRJ01/HV2M23/designer/DD/<user>/HV2M23/DV_TCON_C）
仿真器VCS + UVM-1.1（script/makefile 中 -ntb_opts uvm-1.1），波形 Verdi/FSDB
仿真目录$SIM_PATH/TOP_TCON_C/<case>/（日志 sim.log、golden outResult/、波形 chip_tb_top.vf）
pass/fail 判据base_test.sv report_phase 统计 UVM_ERROR 数，打印 TEST CASE PASSED/FAILED
激励入口所有用例默认只跑 isptx_sequence（见 base_vseq.sv），用例差异全部在配置文件
寄存器配置方式随 ISP 链路数据流写入（CTRL_F/CTRL_L 包）；I2C 仅用于寄存器专项用例
lane 模式支持 1-lane（ONE_PAIR）与 2-lane（TWO_PAIR），cfg_frame 中配置

目录结构

顶层布局

DV_TCON_C/
├── script/                      # 仿真入口与脚本
│   ├── makefile                 # VCS 编译/运行/波形/覆盖率 target
│   ├── run_tc.sh                # 统一运行入口（建仿真目录、调 make）
│   ├── merge_coverage.py        # 合并多用例覆盖率（urg）
│   └── produce_checker1.py
└── top/
    ├── agents/
    │   ├── isprx_env/           # 核心验证环境 source_driver_env（含 cmodel/）
    │   ├── i2c_sv/              # 行为级 I2C master BFM（i2c_model.sv）
    │   └── temp_ymzhu_agent.sv
    ├── tb/
    │   ├── chip_tb_top.sv       # TB 顶层：例化整颗 HV2M23、接口、时钟
    │   ├── base_test.sv         # base_test：env/reg_model/覆盖率组件、pass/fail 判定
    │   ├── intf_connect.sv      # DUT 内部信号 → 观测 interface 的层次化 assign
    │   └── waves_dumper.sv
    ├── vseq/
    │   ├── sd_vsqr.sv           # virtual sequencer
    │   └── base_vseq.sv         # 默认只 uvm_do_on(isptx_seq)
    ├── tests/                   # 日常主战场：t_* 用例目录 + 用例管理脚本
    │   ├── t_8b1lane/           # 基础模板用例
    │   ├── create_new_case.sh   # 新用例生成（按名字解析参数）
    │   ├── run_case.py          # 批量回归（bsub）
    │   ├── check_sim_results.sh # 汇总 PASS/FAIL 到 sim_report.txt
    │   ├── case_list.txt        # 回归用例清单（93 个）
    │   └── ppm_mix_*/           # 共享输入图像（用例 pattern/ 多为 symlink）
    ├── filelist/tb.f            # 环境全部 .sv 文件列表
    ├── regression/hier.cfg      # VCS -cm_hier：只收 digital_top 子树覆盖率
    └── HV2M23_Register_*.xlsx   # 寄存器 spec

关键目录职责

目录职责改动频率
script/仿真调度：makefile 与 run_tc.sh 是所有仿真的唯一入口低
top/tb/TB 骨架：顶层、基类、观测接线，一般不动低
top/agents/isprx_env/验证环境本体：激励、比对、cmodel、env_cfg 自动生成段中（换 spec 时）
top/tests/日常主战场：新建/修改用例、回归清单、共享图像高
top/regression/覆盖率层次配置 hier.cfg低
