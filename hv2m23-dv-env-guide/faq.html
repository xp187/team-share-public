FAQ 与遗留问题

  • makefile CHECK target 失效：里面 grep 的是 xrun.log（历史遗留），实际仿真日志叫 sim.log。
  • merge_coverage.py 硬编码：使用前需确认 project_name/user_names 已改成 HV2M23 和自己的路径。
  • DPLC / DRDOD golden 同步：改 cmodel 后需重新编译 pic_process，并同步到 $DV_TOP_PATH/agents/isprx_env/cmodel/；DRDOD 还需同步 drdod_process.py。
  • 2-lane delay skew：align 相关用例需关注 lane0/lane1 间最大可接受 delay（实测 worst case 约 37~55 UI，与 align 位置有关）。
  • 无独立 checksum checker：HV2M23 没有 HK1V11 那种 standalone checksum checker，寄存器装载由 datapath/I2C 用例间接覆盖。
  • WAKE / CDR ASYNC 行为：CDR ASYNC 对应的信号 rstn_async_L0/L1 行为暂时 keep 1V21 行为，只有 WAKE_PD_OPT=01 不会拉 ASYNC。

新人速查清单

  • 跑一个用例：run_tc.sh TOP t_8b1lane chip_tb_top random clean → 看 $SIM_PATH/TOP_TCON_C/t_8b1lane/sim.log 末尾 TEST CASE PASSED/FAILED。
  • 加用例：cd top/tests && bash create_new_case.sh t_H1920V16_F120_P4_L2_CHSEL1_POLC0_DOTC0_SHL1，再把名字加进 case_list.txt。
  • 定位 fail：check_sim_results.sh 汇总 → sim.log grep UVM_ERROR（scoreboard 报错自带 frame/line/pixel 和期望/实际值）→ run_tc.sh TOP <case> chip_tb_top verdi 看波形；寄存器 dump 在仿真目录 reg_address_frame_*.log。
  • 关键配置开关：用例 .sv 里 6 个 *_check_on（data_merge/digital_top/analog/chopper/drdod_input/drdod_output）；user_def.sv 里 CHIP_SEL/MULTI_CHIP/CONNECT_NUM/DV。
  • 改 golden 行为：cmodel 源码在 cmodel/pic_split_transform/，改完需重新编译 pic_process 并同步；DRDOD 改 drdod_process.py。
  • 波形用例调试：波形用例多在 test class 中 force DUT 内部信号或 sequence 注入错误包，fail 时重点看 LC_CSPIF_L0 状态机与 BCC_I 信号。
